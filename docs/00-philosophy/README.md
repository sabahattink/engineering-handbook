# 00 — Philosophy

This chapter governs how decisions get made and evaluated across every
project in this ecosystem. It does not describe any single project — it
describes the reasoning applied to all of them.

Three documents, each with a different job:

- [`engineering-principles.md`](engineering-principles.md) — the standing
  positions that hold regardless of the situation, and the trade-off each
  one accepts. Read this to understand how a problem gets reasoned about.
- [`decision-framework.md`](decision-framework.md) — the questions,
  evidence, and rejection criteria applied to a specific decision: a new
  project, a dependency, an architectural change, and similar. Read this to
  make a decision.
- [`non-goals.md`](non-goals.md) — what is deliberately not optimized for,
  and what is prioritized instead, so an absence isn't mistaken for an
  oversight. Read this to understand what's intentionally missing.

The principles are standing defaults. The decision framework is how they
get applied to a specific case, including cases where the answer is no. The
non-goals mark the boundary of what this ecosystem is trying to be.
